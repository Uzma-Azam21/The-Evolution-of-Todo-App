# Data Model Specification: Core Backend

**Component**: 001-core-backend  
**Phase**: II  

---

## Entity: Task

### Attributes

- **id**
  - Integer, primary identifier
  - Generated by persistence layer

- **title**
  - Required short text
  - Length constrained

- **description**
  - Optional extended text
  - May be empty or null

- **is_completed**
  - Boolean status flag
  - Defaults to false on creation

- **created_at**
  - Timestamp assigned at creation

- **updated_at**
  - Timestamp refreshed on modification

---

## Behavioral Rules

- Tasks are created in incomplete state
- Updates refresh modification timestamp
- Completion status may toggle freely
- Deleted tasks are permanently removed

---

## Validation Constraints

- Title must be present and non-empty
- Excessively long text values are rejected
- Partial updates must not invalidate entity

---

## Supporting Models

### TaskCreate
- title (required)
- description (optional)

### TaskUpdate
- title (optional)
- description (optional)
- is_completed (optional)

### TaskRead
- Full task representation

### ErrorModel
- detail (string)
- error_code (optional)
